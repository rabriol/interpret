3 Approaches to the Church Translator App

100+ BYOD listeners · 4+ languages · Single room · No internet · Low latency

A

Translator Phone as Wi-Fi Hotspot

Each translator's phone creates a Wi-Fi hotspot. Listeners connect their phones directly to the hotspot for their language. Audio streams over UDP/RTP on the local network.


Pros

  • Zero infrastructure needed
  • Very low latency (~20–40ms)
  • Works on Android & iOS
  • Simple to set up per service

Cons

  • iOS hotspot limited to ~15–50 connections (carrier-dependent)
  • Listeners lose cellular while connected
  • Each language = separate WiFi network to join
  • Android works better as hotspot than iOS

Stack: Flutter · UDP/RTP audio · Native hotspot APIs

B

Dedicated Micro-AP (Raspberry Pi / Travel Router) ⭐ Recommended

A tiny ~$20 device (Raspberry Pi Zero 2W or travel router) acts as a local Wi-Fi access point. All translators and listeners connect to it. Audio distributed via UDP multicast — one stream per language channel. No internet. No router needed.


Pros

  • Scales to 100+ devices reliably
  • Lowest latency (~10–25ms)
  • Single network for all 4+ languages
  • UDP multicast = one stream, N listeners
  • Listeners keep cellular data
  • Portable, fits in a pocket, battery-powered

Cons

  • One small physical device needed
  • One-time setup (~30 min)
  • Listeners must join the Pi's WiFi network

Stack: Flutter · UDP multicast/RTP · Opus codec · Raspberry Pi Zero 2W as AP

C

Bluetooth LE + Platform P2P APIs

Use Apple Multipeer Connectivity (iOS) and Google Nearby Connections (Android) for true device-to-device communication over BLE + Wi-Fi Direct. A bridge device links the two ecosystems.


Pros

  • Truly zero infrastructure
  • Platform-native (fastest integration)
  • No WiFi network switching

Cons

  • iOS and Android can't talk to each other natively
  • Requires a bridge device between ecosystems
  • Max ~8–20 concurrent connections per session
  • BLE audio bandwidth is low quality without pairing
  • Much higher development complexity
  • Does NOT scale to 100+ listeners

Stack: React Native · Multipeer Connectivity · Nearby Connections API · Bridge device

My recommendation: Option B. A Raspberry Pi Zero 2W (~$20, runs on a small USB battery bank) gives you the best of all worlds — no internet, no commercial infrastructure, 100+ devices, lowest latency, and one app for all languages. It's minimal hardware that stays at the church and just works.
