System Design — Church Translator App

Option B · Flutter · Raspberry Pi Zero 2W · UDP/RTP · Opus

System Components

📱 Translator App
  • Selects language channel
  • Captures microphone audio
  • Encodes with Opus (10ms frames)
  • Sends RTP/UDP packets to Pi
  • Push-to-talk OR always-on mode
  • Visual level meter
💾 Pi Relay Server
  • Hosts Wi-Fi AP (hostapd)
  • DHCP server (dnsmasq)
  • Receives UDP per channel
  • Rebroadcasts via multicast
  • Channel registry (mDNS/UDP)
  • No internet, no cloud
🎧 Listener App
  • Fetches available channels
  • Selects language
  • Joins multicast group
  • Decodes Opus frames
  • Plays audio via earphones
  • Volume control

Data Flow

Translator mic ──Opus encode──► RTP/UDP packet ──► Pi:500{n} (unicast)
Pi relay ──rebroadcast──► 239.0.0.{n}:600{n} (multicast)
Listener ──join multicast group──► decode Opus ──► earphone 🎧
Channel 1 (ES): Pi:5001 → multicast 239.0.0.1:6001
Channel 2 (PT): Pi:5002 → multicast 239.0.0.2:6002
Channel N: Pi:500N → multicast 239.0.0.N:600N

App Modes — One Codebase, Two Roles

Single Flutter App
On first launch, user picks role:
🎙️ I am a Translator
🎧 I am a Listener
Role stored locally. Can be changed in settings.
Wi-Fi Connection UX
App detects if connected to ChurchTranslator network.
If not connected, shows a simple prompt: "Join ChurchTranslator Wi-Fi to begin."
No login, no account, no pairing needed.

Pi Setup — One-Time, ~30 Minutes

1. Flash Raspberry Pi OS Lite to SD card
2. Run setup script → configures hostapd (AP), dnsmasq (DHCP), relay server as systemd service
3. Pi boots → broadcasts ChurchTranslator Wi-Fi automatically
4. Every Sunday: plug in USB battery → wait 30 seconds → ready ✅
All config is in a single config.yaml: SSID, password, channel names, number of languages.
